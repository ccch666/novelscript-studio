# Product

## Register

product

## Users

NovelScript Studio is for novel authors, screenwriters, short drama creators, and content studio assistants who need to turn long-form fiction into a structured screenplay draft quickly. During the XEngineer review, judges should be able to understand the product path without reading the code: input at least 3 chapters, generate YAML, inspect scenes, validate schema, edit, and export.

## Product Purpose

The product converts 3 or more chapters of novel text into a schema-validated YAML screenplay draft. Success means the output is not just readable prose, but a structured artifact with characters, locations, scenes, beats, source references, adaptation notes, validation state, and an export path that can be edited further.

## Brand Personality

Calm, precise, and premium. The interface should feel like a focused creative workbench: confident enough for a technical judge, approachable enough for an author, and polished enough to feel like a finished product rather than a demo scaffold.

## Anti-references

Avoid generic AI-purple gradients, decorative dashboards, loud marketing hero layouts, cluttered form screens, nested cards, unreadable low-contrast gray text, and interfaces where the YAML requirement is hidden below generic chat UI. The product must not look like a chatbot wrapper.

## Design Principles

1. Keep the official task visible: 3 chapters, YAML, Schema validation, editable draft, export.
2. Let the workbench show real structure before decoration: chapters, characters, scenes, metrics, and validation state.
3. Optimize the judge path: sample data and complete output should be reachable immediately.
4. Make failures legible: missing backend, invalid chapter count, generation errors, and schema errors must each have clear states.
5. Preserve author trust: the UI should feel stable, readable, and quiet while handling long text and structured output.

## Accessibility & Inclusion

Default to WCAG AA contrast, keyboard-visible focus states, readable Chinese copy, reduced-motion support, and responsive layouts that do not overflow on mobile review screens.
